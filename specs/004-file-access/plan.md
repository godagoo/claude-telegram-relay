# Implementation Plan: File Access for Telegram Interface

**Branch**: `004-file-access` | **Date**: 2026-02-15 | **Spec**: `specs/004-file-access/spec.md`

## Summary

Add read-only file browsing, content injection, and search across two named roots (`share` =
`/mnt/PersonalAssistantHub`, `brain` = SecondBrain data dir) via three Telegram commands
(`/files`, `/read`, `/search`). Add a filesystem watcher for each root: the share watcher
polls via `setInterval`+`stat` (CIFS has no inotify support) and notifies on external changes;
the brain watcher uses `fs.watch` (inotify) to silently trigger scanner re-indexing.

## Design Notes

### FileService

- Constructor takes `FilesConfig`, `Logger`
- `parseRoot(rawPath)` splits first path segment into root name + remainder
- `resolveSafe(root, subpath)` applies `path.resolve` + startsWith containment check
- `isBinary(buf: Buffer)` checks first 8 KB for null bytes; falls back to extension list
- `pendingWrites: Set<string>` is the shared reference passed to `WatcherService`

### WatcherService

- **Share watcher**: `setInterval` at `sharePollIntervalMs`. Each tick:
  1. Walk share root with `readdir`+`stat` recursively (breadth-first, hard depth limit 10 —
     subdirectories beyond depth 10 are skipped with a `log.warn`; no config option needed)
  2. Diff against `shareSnapshot`
  3. Filter changed paths against `pendingWrites`
  4. If non-empty diff remains → call `onShareChange(diff)`
  5. Update snapshot
- **Brain watcher**: `fs.watch(brainRoot, { recursive: true })`. Any event resets debounce timer.
  When timer fires → call `onBrainChange()`.

### Command Registration (index.ts)

```typescript
// Pattern mirrors existing SecondBrain command registration
if (config.files?.shareRoot || config.files?.brainRoot) {
  bot.command("files", (ctx) => handleFiles(ctx, fileService, log));
  bot.command("read", (ctx) => handleRead(ctx, fileService, log));
  bot.command("search", (ctx) => handleSearch(ctx, fileService, log));
  watcherService.start();
  log.info("File access commands registered");
}
```
